IPython Notebook¶

Fernando Pérez, Brian E. Granger, IPython: A System for Interactive Scientific Computing, Computing in Science and Engineering, vol. 9, no. 3, pp. 21-29, May/June 2007, doi:10.1109/MCSE.2007.53. URL: http://ipython.org

Introduction¶

The *IPython Notebook* as you see used here (and throughout this course) is web browser based development environment (DE), which is of special use and interest in scientific computing.

The *Notebook* is not the only DE in the *IPython* family. There is also an interactive shell. It looks like a terminal window and is called the *qt-console*. If you installed a program such as *Anacoda*, you also get a scientific DE, called *Spyder-app*.

The essentials¶

Running IPython¶

Throughout this course, I will be using the *IPython Notebook*. It runs in your default browser and works well in most.

If your system is set up correctly, you can open a *Terminal* window (Linux™; and Mac®) or a *Command* window (Windows®)

Simply type the text: *ipython notebook* and hit enter / return. Your default browser will open up and you will be greeted with a web page that looks like a simple *folder explorer*.

Using an up-to-date *Anaconda* version with *python*™ 3.4 and up, it will give you the ability to navigate your computer folder structure.

I prefer using a simple *Dropbox*® account on my computer to store all my *IPython* files. More sophisticated users can look into other code repositories such as *Github*.

Opening a new notebook¶

Depending on where you navigate to, any new notebook opened will be saved in that folder.

Toolbar¶

Up top you will find a toolbar, with a name for your current notebook.

Simply clicking on the *Untitled0* (the zero can be *1*, *2*, etc. depending on how many notebooks are saved with the *Untitled* name) will allow you to save a name for the notebook.

All the commands are hidden in drop boxes and includes the familiar *File*, *Edit*, and some others. You will quickly learn what these do during the course of the lectures.

Cells¶

The best part of the *IPython Notebook* is the cell. A cell can be converted into a code block or used for text formatting. You can also use CSS code to change the overall look of your notebook as I have done here.

Text can be formatted using markdown or HTML.

Executing a cell is achieved by hitting both the *Shift* and *Enter / Return* keys.

NBConvert¶

Static HTML versions of pages can be rendered in the *Terminal* or *Command* window using a program called *nbconvert*. This way you can get hard copies of your notebooks.

Some simple coding¶

A fancy calculator¶

It is customary to start coding with a *"Hello, world!"* example. I'd rather start with some mathematics.

One of the basic building block of any computer programming language is the *variable*. I'm going to call it a *computer variable* (I also do lectures on mathematics using python™ and I need to discriminate between a computer and an algebraic variable!)

A computer variable is in essence a space in the computer's memory, let's say an empty bucket. The variable name is the name given to that space (bucket). We can put a variety of things in that bucket and manipulate it.

Let's give it a go. I am going to create the variable *a* and put the value *7* into it. Remember, to execute the code in the cell, we hit the *shift* and *enter / return* keys at the same time. Hitting the *enter / return* key just gets you to a new line.
